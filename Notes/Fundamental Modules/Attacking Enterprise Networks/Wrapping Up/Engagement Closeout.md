## Engagement Closeout

At the end of the engagement, the client should be told that testing has ended and when to expect the report. If needed, a report review meeting can be arranged later, but the first priority is clear closure.

## Attack Path Recap

Writing out the attack path from start to finish helps show the path taken and the findings involved. It should show the path of least resistance, not every dead end or the full thought process.

Some firms write reports in narrative form, while others prefer a step by step walkthrough. Either way, this recap helps with reporting and also helps if the client later asks how a specific result was achieved.

## Structuring Findings

Findings should be recorded while testing is still in progress. The notes should stay structured so the content can be moved into the report easily.

If this was not done during testing, the tester should still gather a prioritized finding list, command output, and screenshots before access is lost. The findings list should be ordered from highest to lowest risk.

For more on note taking and reporting, see the [Documentation & Reporting module](https://academy.hackthebox.com/module/162/section/1533). The module also recommends using the sample report from that section and writing a report based on this network.

## Post-Engagement Cleanup

A real engagement requires noting down every scan, attack attempt, file placed on a system, and any change made to the environment. Before the engagement closes, uploaded files should be deleted and the environment should be restored to its original state.

The report appendices should still include every change, uploaded file, account compromise, and host compromise, along with the methods used. Logs and activity records should also be kept for a period after the assessment in case the client needs to correlate alerts.

The text says to treat this module like a real customer network. It also says to go through the lab a second time, perform only minimally invasive actions, and clean up afterward.

## Client Communication

The client must be told when testing has ended so they know any unusual activity is no longer related to the assessment. Communication should continue during the reporting phase, including the report delivery date and a brief recap if the client asks for one.

The project manager may also schedule a report review meeting. If retesting is part of the scope, the client should be given a timeline for remediation and post-remediation testing.

## Internal Closeout

After the report is delivered and the closeout meeting is complete, the company may archive the report and project data, hold a lessons learned debrief, complete a post engagement questionnaire, and handle invoicing. If another tester needs to handle retesting, a knowledge transfer may be needed.
