## Intro to Attacking Enterprise Networks

### Context and Purpose

This module marks the end of the [Penetration Tester Job Role Path](https://academy.hackthebox.com/path/preview/penetration-tester). The path spans 27 modules built around a simulated penetration test against the fictional organisation Inlanefreight, delivered phase by phase and tool by tool. The structure follows the core stages of the [penetration testing process](https://www.pentest-standard.org/index.php/Main_Page), with each module focused on specific Tactics, Techniques, and Procedures (TTPs) required to perform engagements at an intermediate level.

### Penetration Testing Stages Covered

- Pre-Engagement
- Information Gathering
- Vulnerability Assessment
- Exploitation
- Post-Exploitation
- Lateral Movement
- Proof of Concept
- Post-Engagement

### What You Have Completed So Far

By reaching this point, the path has already walked through seven mini simulated penetration tests:

1. All elements of a full pentest split across the 27 preceding modules
2. A step-by-step [Active Directory](https://learn.microsoft.com/en-us/windows-server/identity/ad-ds/get-started/virtual-dc/active-directory-domain-services-overview) pentest in the Active Directory Enumeration and Attacks module sections
3. Two mini simulated Active Directory pentests in the Active Directory Enumeration and Attacks skills assessments
4. One mini simulated pentest in the [Shells and Payloads](https://academy.hackthebox.com/module/details/115) skills assessment
5. One mini simulated pentest in the [Pivoting, Tunneling, and Port Forwarding](https://academy.hackthebox.com/module/details/158) skills assessment
6. One mini simulated internal pentest in the Documentation and Reporting module

Across all modules, over 200 targets have been attacked, covering Windows, Linux, web, and Active Directory environments.

### What This Module Does Differently

Previous modules covered topics in isolation or in limited combinations. This module requires drawing on all knowledge gained across the entire path at once. The workflow will shift between information gathering, web attacks, network attacks, Active Directory attacks, privilege escalation, pillaging, lateral movement, and pivoting without a fixed order.

Every network is unique, but the approach always returns to the core stages of the penetration testing process. Each penetration tester develops their own methodology, but those core stages remain constant.

### Module Structure

This module simulates a full external penetration test against Inlanefreight, which then transitions into a full-scope internal penetration test once access is gained. The approach is structured as a guided "ride-along" with an experienced tester, walking through each step including dead ends and the reasoning behind decisions made along the way.

### Recommended Approach

- Follow the guided walkthrough to complete the lab and submit each flag
- Repeat the lab a second time without the walkthrough, operating as the lead tester
- Take detailed notes throughout, following the documentation practices covered in the [Documentation and Reporting](https://academy.hackthebox.com/module/details/162) module
- Use the second run as practice for producing a commercial-grade penetration testing report
