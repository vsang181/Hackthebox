# Windows Fundamentals

Windows is everywhere in corporate environments, from employee workstations to critical servers. During a penetration testing engagement, you will almost certainly interact with one or more Windows hosts, either as a target or as part of your own attack infrastructure.

To work effectively, you need a solid understanding of **how Windows is structured**, **how its file system works**, and **how to use the command line efficiently**. Without this foundation, tasks such as enumeration, privilege escalation, lateral movement, and post-exploitation become slow, noisy, or unreliable.

Windows is also commonly used as an **attack box**. Many organisations standardise on Windows due to its user-friendly interface and the powerful centralised management provided by **Active Directory**. As a result, you should be comfortable switching roles between attacker and administrator within Windows environments.

In this module, you focus on the **core building blocks** of the Windows operating system. You learn how to navigate the system, interact with it through the command line, and understand the components you will later analyse and exploit. This is not about memorising commands, but about building familiarity so that Windows stops feeling opaque and starts feeling predictable.
