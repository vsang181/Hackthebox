# Web Requests

This module introduces **core fundamentals** that you must understand before you can effectively analyse, attack, or secure any web application. At the heart of every web application interaction is a **web request**, and without a clear understanding of how requests are built, sent, and processed, everything else in web security will feel abstract and confusing.

Before you think about vulnerabilities, exploits, or tools, you need to understand **how the browser and server actually talk to each other**. That is why this module is considered the **starting point for web application penetration testing**.
